## MODIFIED Requirements

### Requirement: Resource Monitor card remains visible without obscuring the stage
The client UI SHALL render a Resource Monitor card that remains visible in both the lobby and in-meeting views while being positioned so it does not obscure the primary stage content in the large-room layout.

#### Scenario: Card visible during large-room meeting
- **WHEN** the user is in an active meeting using the scalable layout
- **THEN** the Resource Monitor card remains visible in meeting chrome outside the primary stage focus area
